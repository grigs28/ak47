# PDF 信息提取优化方案

## 背景

当前流程：所有 PDF 都走"转图片 → 裁图 → qwen-3 视觉识别"提取 6 字段（建设单位、工程名称、设计编号、图名、图号、图别），效率低，每次都要调视觉模型。

实测发现大部分 CAD 矢量 PDF 用 pdfplumber 能直接提取文字，无需视觉模型。只有纯图片 PDF 和纯矢量图形 PDF 才需要走视觉识别。

## 实测数据

| 文件 | 类型 | 有效字符 | pdfplumber 能读 |
|------|------|---------|----------------|
| 00a42745...pdf | CAD 矢量 | 240 | 能，6/6 字段 |
| 01d35e62...pdf | CAD 矢量 | 269 | 能，6/6 字段 |
| 0a9c7fcf...pdf | CAD 矢量 | 498 | 能，6/6 字段 |
| 建筑设计说明...pdf | CAD 矢量 | 402 | 能，6/6 字段 |
| 朔州临建隔离方舱.pdf | 纯图片 | 0 | 不能 |
| 建施_B01_做法表.pdf | 纯矢量图形 | 0 | 不能 |

结论：约 70-80% 的 PDF 可以直接文本提取，只有少数需要走视觉识别。

## 新流程

```
PDF 文件
  │
  ├─ 步骤1：pdfplumber 提取第一页文本
  │
  ├─ 有效字符 > 0？
  │   ├─ 是 → 步骤2：文本提取路径
  │   └─ 否 → 步骤5：视觉识别路径
  │
  ├─ 步骤2：去空格后匹配 6 字段标签
  │   │  （建设单位、工程名称、设计编号、图名、图号、图别）
  │   │  注意："图 名"、"图 号"、"图 别" 中间有空格
  │   │  解决：compact = text.replace(' ', '').replace('\u3000', '')
  │   │
  │   ├─ 6 字段标签不全 → pass（不是标准图签格式）
  │   └─ 6 字段标签全 → 步骤3：提取字段值
  │
  ├─ 步骤3：提取字段值
  │   │  用正则匹配 "字段名:值" 或 "字段名：值"
  │   │  值可以为空
  │   │  示例：compact 中 "设计编号:S25-258-1"
  │   │        → 设计编号 = "S25-258-1"
  │   │
  │   └─ 步骤4：判断是否说明
  │       │  去空格后模糊匹配关键词：
  │       │  "设计说明"、"总说明"、"设计总说明"
  │       │
  │       └─ 完成，返回结果
  │
  └─ 步骤5：视觉识别路径（兜底）
      │
      ├─ PDF 第一页转图片（pdftoppm, dpi=200）
      │
      ├─ 判断横竖：宽>高=横幅，高>宽=竖幅
      │
      ├─ 裁图策略（20%）：
      │   ├─ 横幅：裁右侧 20%，占满高度
      │   └─ 竖幅：先裁上部 20% 占满宽度，再裁下部 20% 占满宽度
      │
      └─ 裁图送 qwen-3 提取 6 字段 + 判断是否说明
          └─ 6 字段不全 → pass
```

## 需要修改的文件

### 1. app/vision/extractor.py（主要修改）

**现状**：`extract()` 方法直接转图片 → 裁图 → qwen-3

**改为**：
- 新增 `_extract_from_text(pdf_path)` 方法
  - 用 pdfplumber 读取第一页文本
  - 去空格后检查 6 字段标签是否齐全
  - 用正则提取字段值
  - 模糊匹配判断是否"说明"
  - 返回统一格式的结果 dict

- 修改 `extract()` 主方法流程：
  ```
  1. 先调 _extract_from_text()
  2. 成功 → 返回结果
  3. 失败（无文字/字段不全）→ 走原有视觉识别流程
  ```

### 2. app/vision/utils.py（修改裁图策略）

**现状**：`get_crop_strategy()` 竖版先裁底部再裁顶部

**改为**：
- 竖幅：先裁上部 → 再裁下部（图签在上部更常见）
- 横幅：裁右侧（不变）

### 3. app/vision/classifier.py（可能调整）

- 文本路径已经能判断是否说明，不需要额外调 classifier
- 视觉路径不变

### 4. app/scan.py（可能微调）

- `_process_single_pdf_thread_safe()` 中调用 extractor 的逻辑不变
- extractor 内部自动选择文本/视觉路径

## 正则提取规则

```python
# 去空格后的文本中匹配
patterns = {
    '建设单位': r'建设单位[：:]\s*(.*?)(?=工程名称|设计编号|图名|图号|图别|$)',
    '工程名称': r'工程名称[：:]\s*(.*?)(?=建设单位|设计编号|图名|图号|图别|$)',
    '设计编号': r'设计编号[：:]\s*([\w\-]+)',
    '图名':     r'图名[：:]\s*(.*?)(?=图号|图别|建设单位|$)',
    '图号':     r'图号[：:]\s*(\S+)',
    '图别':     r'图别[：:]\s*(\S+)',
}
```

注意：pdfplumber 提取的文本中，字段和值可能跨行，需要处理换行情况。

## 说明判断规则

```python
# 去空格后模糊匹配
instruction_keywords = ['设计说明', '总说明', '设计总说明', '施工图设计总说明']
compact = text.replace(' ', '').replace('\u3000', '').replace('\n', '')
is_instruction = any(kw in compact for kw in instruction_keywords)
```

## 性能预期

| 路径 | 耗时 | 适用比例 |
|------|------|---------|
| pdfplumber 文本提取 | <100ms | ~80% |
| qwen-3 视觉识别 | 3-10s | ~20% |

整体速度提升约 4-5 倍。
